## 2. Algorithm Design: watermark-diagonal.ts

### Problem Statement

Stock photo services (Shutterstock, Getty, iStock, Adobe Stock, Alamy) overlay diagonal repeating text watermarks across the entire image. These are semi-transparent text strings rotated ~30-45 degrees, tiled in a grid pattern. The text is typically light gray or white with low opacity over the image.

### Algorithm Overview

The detector uses a 3-stage pipeline: (1) Sobel gradient extraction to isolate edge information, (2) diagonal projection to accumulate energy along diagonal strips, (3) periodicity analysis via autocorrelation to confirm repeating pattern.

### Detailed Pseudocode

```
function watermarkDetectDiagonal(
  pixels: Uint8ClampedArray,
  width: number,
  height: number,
): WatermarkResult

  // ── Stage 1: Grayscale + Sobel Gradient Magnitude ──
  // Convert to grayscale: gray[i] = 0.299*R + 0.587*G + 0.114*B
  // Apply 3x3 Sobel kernels:
  //   Gx = [[-1,0,1],[-2,0,2],[-1,0,1]]
  //   Gy = [[-1,-2,-1],[0,0,0],[1,2,1]]
  // magnitude[y][x] = sqrt(Gx^2 + Gy^2)
  // Normalize magnitude to 0-255 range

  // ── Stage 2: Diagonal Strip Projection ──
  // For each candidate angle theta in [30, 35, 40, 45] degrees:
  //   Project gradient magnitude along strips perpendicular to theta.
  //   A "strip" is a 1px-wide band at angle (theta+90).
  //
  //   For each pixel (x, y):
  //     stripIndex = round(x * sin(theta) - y * cos(theta))
  //
  //   Accumulate gradient magnitude into strip bins:
  //     stripEnergy[stripIndex] += magnitude[y][x]
  //     stripCount[stripIndex] += 1
  //
  //   Normalize: stripAvg[i] = stripEnergy[i] / stripCount[i]

  // ── Stage 3: Periodicity Detection via Autocorrelation ──
  // For each angle's stripAvg:
  //   Compute autocorrelation R(lag) for lag in [MIN_PERIOD..MAX_PERIOD]:
  //     R(lag) = sum( (stripAvg[i] - mean) * (stripAvg[i+lag] - mean) ) / variance
  //
  //   Find peaks in R(lag) above PERIODICITY_THRESHOLD.
  //   If >= MIN_PERIOD_PEAKS peaks found at consistent spacing:
  //     This angle has a periodic diagonal pattern.
  //     Record bestAngle = theta, bestPeriod = dominant lag.

  // ── Stage 4: Distinguish Watermark from Real Content ──
  // Real diagonal content (striped shirts, fences, blinds) differs from
  // watermarks in key ways:
  //
  // a) Spatial extent: watermarks tile across the FULL image (>70% coverage).
  //    Real diagonals are localized. Check: what fraction of strips have
  //    above-average energy? If < COVERAGE_THRESHOLD (0.6), reject.
  //
  // b) Periodicity regularity: watermark periods are extremely consistent
  //    (std of inter-peak distances < REGULARITY_THRESHOLD). Real content
  //    has irregular spacing.
  //
  // c) Gradient magnitude profile: watermark text creates a specific
  //    signature — thin edges with gaps between letters. Compute the
  //    coefficient of variation (std/mean) of strip energies within one
  //    period. Watermarks have CV in [WATERMARK_CV_MIN..WATERMARK_CV_MAX].

  if no periodic angle found:
    return { detected: false, mask: null }

  // ── Stage 5: Mask Generation ──
  // For detected angle and period:
  //   For each pixel (x, y):
  //     stripIndex = round(x * sin(bestAngle) - y * cos(bestAngle))
  //     Compute local gradient magnitude
  //     If magnitude > EDGE_THRESHOLD AND strip is in a "watermark band"
  //       (stripIndex mod period falls within text portion):
  //       mask[y * width + x] = 1
  //
  //   Dilate mask by DILATION_RADIUS to cover anti-aliased text edges.
  //   This uses a simple box dilation: for each masked pixel, mark
  //   all neighbors within DILATION_RADIUS as masked.

  return { detected: true, mask, angle: bestAngle, period: bestPeriod }
```

### Constants (added to `DIAGONAL_WATERMARK_PARAMS`)

```typescript
export const DIAGONAL_WATERMARK_PARAMS = {
  /** Candidate angles in degrees to test */
  CANDIDATE_ANGLES: [30, 35, 40, 45] as const,
  /** Minimum strip period (pixels) for valid watermark repetition */
  MIN_PERIOD: 60,
  /** Maximum strip period (pixels) */
  MAX_PERIOD: 400,
  /** Autocorrelation threshold to count as a peak */
  PERIODICITY_THRESHOLD: 0.3,
  /** Minimum number of autocorrelation peaks to confirm periodicity */
  MIN_PERIOD_PEAKS: 3,
  /** Minimum fraction of strips with above-average energy (full-image tiling) */
  COVERAGE_THRESHOLD: 0.6,
  /** Max std of inter-peak distances (regularity check) */
  REGULARITY_THRESHOLD: 0.15,
  /** Coefficient of variation range for watermark text signature */
  WATERMARK_CV_MIN: 0.3,
  WATERMARK_CV_MAX: 1.5,
  /** Gradient magnitude threshold for edge detection in mask */
  EDGE_THRESHOLD: 30,
  /** Dilation radius for mask expansion (pixels) */
  DILATION_RADIUS: 3,
  /** Downsample factor for performance (process at 1/N resolution) */
  DOWNSAMPLE: 2,
} as const;
```

### Performance Considerations

- **Downsample by 2x** before Sobel: watermark text features are large enough to survive halving. This cuts pixel count by 4x.
- Sobel is O(W\*H) — fast for typical image sizes.
- Autocorrelation is O(strips \* maxPeriod) — manageable.
- Total expected runtime: <100ms for a 2048x2048 image.

### Function Signature

```typescript
export function watermarkDetectDiagonal(
  pixels: Uint8ClampedArray,
  width: number,
  height: number,
): WatermarkResult;
```

Note: unlike the Gemini detector, this does NOT need background colors as input — it operates purely on gradient information.

## 3. Algorithm Design: watermark-corner.ts

### Problem Statement

AI tools (Adobe Firefly, Canva, etc.) and some stock sites place small logos or badges in image corners. Unlike the Gemini sparkle detector (which only checks bottom-right and uses deviation from bg colors), we need a generalized detector that:

- Scans all 4 corners
- Does NOT depend on knowing the background color
- Handles both opaque logos and semi-transparent overlays
- Distinguishes logos from actual image subject content near corners

### How It Differs from Gemini Sparkle Detector

| Aspect               | Gemini (`watermark-detect.ts`) | Corner (`watermark-corner.ts`)         |
| -------------------- | ------------------------------ | -------------------------------------- |
| Corners scanned      | Bottom-right only              | All 4                                  |
| Depends on bg colors | Yes (colorA, colorB)           | No                                     |
| Detection method     | Pixel deviation from bg        | Cluster analysis of local anomalies    |
| Target               | Single sparkle glyph           | Rectangular logo/badge of varying size |
| Size range           | ~10-30px radius                | 20-120px per side                      |

### Algorithm Overview

For each of the 4 corners, extract a region, compute local statistics, and look for a cluster of pixels that deviate from the surrounding area. Use spatial coherence and size heuristics to distinguish logos from subject matter.

### Detailed Pseudocode

```
function watermarkDetectCorner(
  pixels: Uint8ClampedArray,
  width: number,
  height: number,
): WatermarkResult

  // ── Configuration ──
  // scanSize = max(CORNER_MIN_SCAN, min(width, height) / CORNER_SCAN_FRACTION)
  // scanSize = min(scanSize, min(width, height) / 3)  // never more than 1/3

  // ── For each corner: TL, TR, BL, BR ──
  for corner in [TOP_LEFT, TOP_RIGHT, BOTTOM_LEFT, BOTTOM_RIGHT]:

    // Extract corner region coordinates
    // TL: (0, 0) to (scanSize, scanSize)
    // TR: (width-scanSize, 0) to (width, scanSize)
    // BL: (0, height-scanSize) to (scanSize, height)
    // BR: (width-scanSize, height-scanSize) to (width, height)

    // ── Step 1: Compute local background estimate ──
    // Sample a 1-pixel-wide border strip along the two edges of the
    // corner that face inward (e.g., for TL: the right edge and bottom
    // edge of the scan region). These strips are likely to be background
    // because logos sit INSIDE the corner, not at its inner edges.
    //
    // Compute median RGB of border strip = estimatedBg

    // ── Step 2: Compute deviation map ──
    // For each pixel (x, y) in the corner region:
    //   deviation[ly][lx] = euclideanDist(pixel[x,y], estimatedBg)
    //     where euclideanDist = sqrt((dR^2 + dG^2 + dB^2) / 3)

    // ── Step 3: Binary threshold ──
    // Compute Otsu threshold on the deviation map to separate
    // "background-like" pixels from "anomalous" pixels.
    // deviationMask[ly][lx] = deviation[ly][lx] > otsuThreshold ? 1 : 0

    // ── Step 4: Connected component analysis ──
    // Find connected components in deviationMask using 4-connectivity BFS.
    // For each component, compute:
    //   - area (pixel count)
    //   - bounding box (minX, minY, maxX, maxY)
    //   - aspect ratio of bounding box
    //   - distance from corner point

    // ── Step 5: Logo candidate filtering ──
    // A connected component is a logo candidate if:
    //   a) area >= MIN_LOGO_AREA and area <= MAX_LOGO_AREA
    //   b) bounding box fits within [MIN_LOGO_DIM..MAX_LOGO_DIM] range
    //   c) aspect ratio between LOGO_ASPECT_MIN and LOGO_ASPECT_MAX
    //   d) centroid is within CORNER_PROXIMITY of the actual corner point
    //      (logos hug the corner, subjects tend toward center)
    //   e) component does NOT touch the inner edges of the scan region
    //      (a subject extending from the image center would touch inner
    //      edges; a corner logo would not)

    // ── Step 6: Aggregate candidates ──
    // If one or more logo candidates pass all filters:
    //   Merge their bounding boxes.
    //   Mark this corner as detected.

  // ── Build combined mask ──
  // For each detected corner:
  //   Create rectangular mask covering merged bounding box + MASK_PADDING.
  //   Fill the mask with 1s for all pixels within that padded rectangle.

  // If no corners detected: return { detected: false, mask: null }

  return {
    detected: true,
    mask: combinedMask,
    cornersDetected: [...],  // for debugging
  }
```

### Constants (added to `CORNER_WATERMARK_PARAMS`)

```typescript
export const CORNER_WATERMARK_PARAMS = {
  /** Minimum scan region per corner (pixels) */
  CORNER_MIN_SCAN: 80,
  /** Fraction of min(W,H) for scan region */
  CORNER_SCAN_FRACTION: 6,
  /** Minimum logo area in pixels */
  MIN_LOGO_AREA: 100,
  /** Maximum logo area in pixels (avoid flagging subjects) */
  MAX_LOGO_AREA: 15_000,
  /** Minimum logo dimension (bounding box side) */
  MIN_LOGO_DIM: 10,
  /** Maximum logo dimension */
  MAX_LOGO_DIM: 120,
  /** Logo bounding box aspect ratio range */
  LOGO_ASPECT_MIN: 0.2,
  LOGO_ASPECT_MAX: 5.0,
  /** Max distance from corner point to logo centroid (fraction of scanSize) */
  CORNER_PROXIMITY: 0.7,
  /** Padding around detected logo for mask (pixels) */
  MASK_PADDING: 5,
  /** Minimum deviation (Euclidean color distance) for Otsu fallback */
  MIN_DEVIATION_THRESHOLD: 15,
} as const;
```

### Why Not Use Sobel Here?

Logos are typically solid-colored or gradient-filled regions, not edge-heavy structures. The Euclidean color deviation from local background is more robust for detecting filled shapes than edge detection would be. Sobel would find the logo's outline but miss its interior fill.

### Function Signature

```typescript
export function watermarkDetectCorner(
  pixels: Uint8ClampedArray,
  width: number,
  height: number,
): WatermarkResult;
```

No color inputs needed — the algorithm estimates local background per corner.

## 4. PWA Service Worker Strategy

### Cache Architecture

```
Cache name: nukebg-shell-v{N}    (N = integer, bumped on deploy)
```

**What gets cached (app shell):**

| Asset                           | Strategy               | Reason                                               |
| ------------------------------- | ---------------------- | ---------------------------------------------------- |
| `/` (index.html)                | Stale-while-revalidate | Always serve cached page, fetch update in background |
| `/assets/*.js`                  | Cache-first            | Vite hashes filenames; immutable                     |
| `/assets/*.css`                 | Cache-first            | Same — content-hashed                                |
| `/fonts/*.woff2`                | Cache-first            | Static, never changes                                |
| `/favicon*.png`, `/favicon.svg` | Cache-first            | Static                                               |
| `/apple-touch-icon.png`         | Cache-first            | Static                                               |
| `/og-image.png`                 | Cache-first            | Static                                               |
| `/manifest.webmanifest`         | Stale-while-revalidate | May change on version bump                           |
| `/icon-*.png`                   | Cache-first            | Static PWA icons                                     |

**What is NEVER cached:**

| URL Pattern                 | Reason                                             |
| --------------------------- | -------------------------------------------------- |
| `*huggingface.co*`          | 45MB RMBG model — let browser HTTP cache handle it |
| `*cdn.jsdelivr.net*`        | Transformers.js CDN — same reason                  |
| `*analytics*`, `*tracking*` | Should not exist but defensive exclusion           |

### Service Worker Pseudocode

```javascript
const CACHE_NAME = 'nukebg-shell-v3';
const SHELL_URLS = [
  '/',
  // Vite-hashed assets are added dynamically during build via
  // a Vite plugin or manual __WB_MANIFEST-like approach.
  // For now, we cache on first fetch (runtime caching).
];

// URLs that must NEVER be cached
const NEVER_CACHE_PATTERNS = [
  /huggingface\.co/,
  /cdn\.jsdelivr\.net/,
  /onnxruntime/,
];

function shouldCache(url) {
  return NEVER_CACHE_PATTERNS.every((pattern) => !pattern.test(url));
}

// Install: pre-cache critical shell
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches
      .open(CACHE_NAME)
      .then((cache) => cache.addAll(SHELL_URLS))
      .then(() => self.skipWaiting()),
  );
});

// Activate: clean old caches
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches
      .keys()
      .then((keys) =>
        Promise.all(
          keys.filter((k) => k !== CACHE_NAME).map((k) => caches.delete(k)),
        ),
      )
      .then(() => self.clients.claim()),
  );
});

// Fetch: stale-while-revalidate for navigation,
//        cache-first for hashed assets
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url);

  // Never cache excluded URLs
  if (!shouldCache(url.href)) return;

  // Only handle same-origin GET requests
  if (event.request.method !== 'GET') return;
  if (url.origin !== self.location.origin) return;

  // Navigation requests (HTML): stale-while-revalidate
  if (event.request.mode === 'navigate') {
    event.respondWith(
      caches.open(CACHE_NAME).then((cache) =>
        cache.match('/').then((cached) => {
          const fetchPromise = fetch(event.request).then((response) => {
            if (response.ok) cache.put('/', response.clone());
            return response;
          });
          return cached || fetchPromise;
        }),
      ),
    );
    return;
  }

  // Static assets: cache-first, fallback to network
  event.respondWith(
    caches.match(event.request).then((cached) => {
      if (cached) return cached;
      return fetch(event.request).then((response) => {
        if (response.ok && shouldCache(url.href)) {
          const clone = response.clone();
          caches
            .open(CACHE_NAME)
            .then((cache) => cache.put(event.request, clone));
        }
        return response;
      });
    }),
  );
});

// Message: version check from sw-register.ts
self.addEventListener('message', (event) => {
  if (event.data?.type === 'GET_VERSION') {
    event.ports[0].postMessage({ version: CACHE_NAME });
  }
});
```

### sw-register.ts Rewrite

```typescript
// Key responsibilities:
// 1. Register SW
// 2. Capture beforeinstallprompt for deferred install
// 3. Detect SW updates and notify UI
// 4. Expose installApp() for the install button

let deferredPrompt: BeforeInstallPromptEvent | null = null;
let isInstallable = false;

// Capture install prompt
window.addEventListener('beforeinstallprompt', (e: Event) => {
  e.preventDefault();
  deferredPrompt = e as BeforeInstallPromptEvent;
  isInstallable = true;
  document.dispatchEvent(new CustomEvent('nukebg:install-available'));
});

// Detect when app is installed
window.addEventListener('appinstalled', () => {
  deferredPrompt = null;
  isInstallable = false;
  document.dispatchEvent(new CustomEvent('nukebg:app-installed'));
});

export function canInstall(): boolean {
  return isInstallable;
}

export async function installApp(): Promise<boolean> {
  if (!deferredPrompt) return false;
  deferredPrompt.prompt();
  const result = await deferredPrompt.userChoice;
  deferredPrompt = null;
  isInstallable = false;
  return result.outcome === 'accepted';
}

// Register SW
if ('serviceWorker' in navigator) {
  window.addEventListener('load', async () => {
    try {
      const reg = await navigator.serviceWorker.register('/service-worker.js');

      // Check for updates every 60 minutes
      setInterval(() => reg.update(), 60 * 60 * 1000);

      // Detect waiting SW (new version ready)
      reg.addEventListener('updatefound', () => {
        const newWorker = reg.installing;
        if (!newWorker) return;

        newWorker.addEventListener('statechange', () => {
          if (
            newWorker.state === 'installed' &&
            navigator.serviceWorker.controller
          ) {
            // New version available — notify UI
            document.dispatchEvent(new CustomEvent('nukebg:update-available'));
          }
        });
      });
    } catch (err) {
      console.warn('Service Worker registration failed:', err);
    }
  });
}
```

### Re-export Flow

When the user clicks a format button:

4. Call `this.reExport()`.

```typescript
private async reExport(): Promise<void> {
  if (!this.imageData) return;

  const { exportPng, exportWebp, generateOutputFilename } = await import('../utils/image-io');

  let blob: Blob;
  if (this.selectedFormat === 'webp') {
    blob = await exportWebp(this.imageData);
  } else {
    blob = await exportPng(this.imageData);
  }

  this.resultBlob = blob;
  this.filename = generateOutputFilename(this.inputFilename, this.selectedFormat);

  if (this.blobUrl) URL.revokeObjectURL(this.blobUrl);
  this.blobUrl = URL.createObjectURL(blob);

  this.update();
}
```

### exportWebp() in image-io.ts

```typescript
export async function exportWebp(
  imageData: ImageData,
  quality = 0.9,
): Promise<Blob> {
  if (typeof OffscreenCanvas !== 'undefined') {
    const canvas = new OffscreenCanvas(imageData.width, imageData.height);
    const ctx = canvas.getContext('2d')!;
    ctx.putImageData(imageData, 0, 0);
    return canvas.convertToBlob({ type: 'image/webp', quality });
  } else {
    const canvas = document.createElement('canvas');
    canvas.width = imageData.width;
    canvas.height = imageData.height;
    const ctx = canvas.getContext('2d')!;
    ctx.putImageData(imageData, 0, 0);
    return new Promise<Blob>((resolve, reject) => {
      canvas.toBlob(
        (blob) => {
          if (blob) resolve(blob);
          else reject(new Error('Canvas toBlob failed for WebP'));
        },
        'image/webp',
        quality,
      );
    });
  }
  // Note: No metadata injection for WebP (XMP complexity deferred per spec).
}
```

## 8. Risk Register

| #   | Risk                                                                                                         | Probability | Impact                                      | Mitigation                                                                                                                                                                                                                                                                                                        |
| --- | ------------------------------------------------------------------------------------------------------------ | ----------- | ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| R1  | **Diagonal detector false positives on real diagonal content** (striped shirts, blinds, architectural lines) | Medium      | High (unwanted inpainting destroys image)   | Three-filter defense: coverage check (>60% of strips), regularity check (consistent spacing), CV range check. Conservative thresholds — better to miss a watermark than damage content. Add `contentType === 'ICON'` skip (icons never have stock watermarks).                                                    |
| R2  | **Corner detector flags subject matter as logo** (e.g., person's head near corner)                           | Medium      | High                                        | Inner-edge filter: if the connected component touches the inward-facing edges of the scan region, it is part of the main image subject, not a corner logo. Max area cap prevents large subjects from being flagged.                                                                                               |
| R3  | **WebP export not supported in Safari <16.4**                                                                | Low         | Medium                                      | `canvas.convertToBlob({type: 'image/webp'})` returns an empty blob or throws on unsupported browsers. Detect by checking `blob.size === 0` and fall back to PNG with a toast notification.                                                                                                                        |
| R4  | **Service worker caches stale app shell after deploy**                                                       | Medium      | Medium                                      | Stale-while-revalidate means users always get the cached version first, but the fresh version is fetched in background. On next navigation, they get the update. The `nukebg:update-available` event lets us show a "New version available, refresh" banner. Cache name includes version number for clean purges. |
| R5  | **SW accidentally caches HuggingFace model**                                                                 | Low         | Critical (45MB in Cache API = quota issues) | URL pattern exclusion is the first defense. Additionally, only same-origin requests are cached. HuggingFace URLs are cross-origin, so they are excluded by the `url.origin !== self.location.origin` check. Double protection.                                                                                    |
| R6  | **i18n translations drift** (new keys added in EN but forgotten in other locales)                            | High        | Low                                         | Test `tests/i18n/translations.test.ts` explicitly asserts all locales have the same key set as English. CI will catch missing keys.                                                                                                                                                                               |
| R7  | **Format toggle re-export is slow for large images**                                                         | Low         | Medium                                      | WebP encoding is done by the browser engine (C++ canvas implementation), not JS. For a 4096x4096 image, `convertToBlob` for WebP takes ~200-500ms, which is acceptable. Show a brief loading indicator on the download button during re-export if needed.                                                         |
| R8  | **beforeinstallprompt event never fires** (Chrome-only, not in Firefox/Safari)                               | High        | Low                                         | The install button is progressive enhancement — it only appears when the event fires. No install button shown = no broken UX. Safari users can still "Add to Home Screen" manually.                                                                                                                               |
| R9  | **Diagonal watermark Sobel + autocorrelation is slow on large images**                                       | Medium      | Medium                                      | Downsample by 2x before processing. At 2048x2048 input, we process at 1024x1024 = 1M pixels. Sobel is simple per-pixel math. Autocorrelation operates on ~1400 strip bins with lags up to 400. Total estimate: <100ms. If it exceeds 200ms, increase DOWNSAMPLE to 3 or 4.                                        |
| R10 | **Multiple watermark detectors find overlapping regions, causing over-inpainting**                           | Low         | Medium                                      | The existing `combineMasks` uses OR, so overlapping regions are simply merged. This is correct behavior — the inpainter only sees a binary mask. The Telea FMM inpainter handles large masks gracefully.                                                                                                          |

### Decision Classification

| Decision                                                   | Type                | Reasoning                                                                                                                                              |
| ---------------------------------------------------------- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| WebP as secondary format (PNG stays default)               | Two-way door        | Easy to change default later or add more formats                                                                                                       |
| Stale-while-revalidate SW strategy                         | Two-way door        | SW can be replaced/updated trivially                                                                                                                   |
| Sobel + autocorrelation for diagonal detection             | Two-way door        | Algorithm is a pure function, can swap implementation without touching interfaces                                                                      |
| Corner detector using local bg estimation (no color input) | One-way door (mild) | Sets the API contract. But since `WatermarkResult` is unchanged, the downstream impact is zero.                                                        |
| Excluding model from SW cache                              | One-way door        | If we later want to cache the model, we would need to redesign quota management. Correct decision for now — 45MB in Cache API is risky for user quota. |

Update `combineMasks` to accept N masks:

```typescript
private static combineMasks(
  masks: (Uint8Array | null)[],
  size: number,
): Uint8Array | null {
  const valid = masks.filter((m): m is Uint8Array => m !== null);
  if (valid.length === 0) return null;
  if (valid.length === 1) return valid[0];

  const combined = new Uint8Array(size);
  for (let i = 0; i < size; i++) {
    for (const mask of valid) {
      if (mask[i]) {
        combined[i] = 1;
        break;
      }
    }
  }
  return combined;
}
```

Call site:

```typescript
const anyWatermark =
  wmGemini.detected ||
  wmDalle.detected ||
  wmDiagonal.detected ||
  wmCorner.detected;
const combinedMask = PipelineOrchestrator.combineMasks(
  [wmGemini.mask, wmDalle.mask, wmDiagonal.mask, wmCorner.mask],
  width * height,
);
```

Note: since all 4 calls go through the same CV worker (single thread), they are queued — not truly parallel at the worker level. However, the `Promise.all` pattern is correct because:

1. The orchestrator sends all 4 messages without waiting.
2. The CV worker processes them sequentially but returns results as they complete.
3. The main thread only blocks on the slowest one.

If performance becomes a concern, we could split into 2 CV workers (one for the quick detectors, one for the Sobel-heavy diagonal). This is a two-way door.

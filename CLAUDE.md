# NukeBG -- Project Conventions

## Description

NukeBG is an open source web tool for creators working with AI-generated images. It removes backgrounds, checkerboard patterns, and watermarks. All processing is 100% client-side in the user's browser. Licensed under GPL-3.0.

## Commands

```bash
# Install dependencies
npm install

# Dev server (http://localhost:5173)
npm run dev

# Run tests (Vitest, single run)
npm test

# Tests in watch mode
npm run test:watch

# Production build (tsc + vite build -> dist/)
npm run build

# Preview production build
npm run preview
```

## Language Conventions

- **Code** (variables, functions, classes, interfaces): **English**
- **Code comments**: **English**
- **All documentation**: **English**
- **Commits**: description in **English**

## Commit Conventions

Format: `type: description`

Valid types: `feat`, `fix`, `docs`, `test`, `refactor`, `infra`, `security`, `design`

Examples:
```
feat: add DALL-E watermark detection
fix: correct flood-fill at image borders
test: add tests for 64px grid checkerboard
infra: configure Cloudflare Pages deploy
```

## Development Rules

### TypeScript
- Strict mode enabled. Never use `any`.
- Shared types live in `src/types/`.
- CV algorithms must be pure functions: data in, result out, no side effects.

### Parameters
- All magic numbers and thresholds go in `src/pipeline/constants.ts`.
- Never hardcode values in algorithm modules.

### Workers
- Web Workers are stateless: receive `ImageData` via `Transferable`, process, return result.
- The Pipeline Orchestrator lives on the main thread and coordinates workers.

### UI
- Native Web Components with Shadow DOM. No framework.
- Inter-component communication via Custom Events on `document`.
- WCAG 2.1 AA required. Support `prefers-reduced-motion`.

### Security
- Strict CSP: no `eval`, no inline scripts.
- SRI on CDN assets.
- No cookies, no tracking, no analytics.

### Principles
- Privacy is non-negotiable: everything client-side, images never leave the device.
- Quality over speed.
- Offline-capable after first load.
- Deployable to static hosting (Cloudflare Pages, GitHub Pages, Netlify, Vercel).

## Tech Stack

- **Runtime**: Vanilla TypeScript + Web Components
- **Build**: Vite 6
- **Tests**: Vitest + happy-dom
- **ML**: Transformers.js with RMBG-1.4 INT8 (~45MB) + MODNet (~25MB), lazy-loaded
- **GPU**: WebGPU with automatic WASM fallback
- **Styles**: Custom CSS (JetBrains Mono, zero dependencies)
- **Cache**: Service Worker + Cache API

# NukeBG -- Project Conventions

## Description

NukeBG is an open source web tool for creators working with AI-generated images. It removes backgrounds, checkerboard patterns, and watermarks using a dual ML pipeline (RMBG-1.4 + ViTMatte). All processing is 100% client-side in the user's browser. Licensed under GPL-3.0.

## Commands

```bash
# Install dependencies
npm install

# Dev server (http://localhost:5173)
npm run dev

# Run tests (Vitest, single run)
npm test

# Tests in watch mode
npm run test:watch

# Production build (tsc + vite build -> dist/)
npm run build

# Preview production build
npm run preview
```

## Pipeline (v2.0)

```
INPUT → [1. Scan] → [2. Watermark detect] → [3. Inpaint if needed] → [4. RMBG-1.4] → [5. ViTMatte] → OUTPUT
         CV instant    CV instant              CV instant               ML ~3-10s       ML ~3-10s
```

1. **Scan image** — corner sampling, classify bg as checkerboard/solid/complex (CV)
2. **Watermark detection** — Gemini sparkle + DALL-E color bar (CV)
3. **Watermark inpainting** — Telea FMM reconstruction, only if watermark detected (CV)
4. **Background removal** — RMBG-1.4 via Transformers.js, produces binary alpha mask (ML)
5. **Edge refinement** — ViTMatte alpha matting, refines edges using trimap from step 4 (ML)

Memory management: RMBG-1.4 worker is terminated before ViTMatte worker starts. Only one ML model in memory at a time.

## Development Rules

### TypeScript
- Strict mode enabled. Never use `any`.
- Shared types live in `src/types/`.
- CV algorithms must be pure functions: data in, result out, no side effects.

### Parameters
- All magic numbers and thresholds go in `src/pipeline/constants.ts`.
- Never hardcode values in algorithm modules.

### Workers
- Web Workers are stateless: receive `ImageData` via `Transferable`, process, return result.
- The Pipeline Orchestrator lives on the main thread and coordinates workers.
- Only one ML model in memory at a time (dispose before loading next).

### UI
- Native Web Components with Shadow DOM. No framework.
- Inter-component communication via Custom Events on `document`.
- WCAG 2.1 AA required. Support `prefers-reduced-motion`.

### Security
- Strict CSP: no `eval`, no inline scripts.
- SRI on CDN assets.
- No cookies, no tracking, no analytics.

### Principles
- Privacy is non-negotiable: everything client-side, images never leave the device.
- Quality over speed.
- Offline-capable after first load.
- Deployable to static hosting (Cloudflare Pages, GitHub Pages, Netlify, Vercel).

## Tech Stack

- **Runtime**: Vanilla TypeScript + Web Components
- **Build**: Vite 6
- **Tests**: Vitest + happy-dom
- **ML**: Transformers.js with RMBG-1.4 INT8 (~45MB) for segmentation, ViTMatte (~25MB) for edge refinement
- **GPU**: WASM (WebGPU reserved for future stability)
- **Styles**: Custom CSS (JetBrains Mono, zero dependencies)
- **Cache**: Service Worker + Cache API
- **i18n**: Lightweight custom system (EN/ES)
